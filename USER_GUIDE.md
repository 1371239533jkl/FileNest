# 智能文件管家 - 使用文档

## 1. 简介

智能文件管家是一款本地运行的AI驱动文件整理工具，帮助你自动扫描、分类、去重和管理文件。所有数据存储在本地MySQL数据库中，无需联网，保障隐私安全。

## 2. 环境要求

- Python 3.9+
- MySQL 5.7+ (运行在 localhost:3306)
- Windows 操作系统

## 3. 安装与启动

### 3.1 安装依赖

```bash
cd smart-file-manager
..\.venv\Scripts\pip.exe install -r requirements.txt
```

依赖包：PyQt6、PyMySQL、Pillow、PyPDF2

### 3.2 配置数据库

在项目根目录创建 `.env` 文件，配置数据库密码：

```ini
# .env
SMART_FM_DB_PASSWORD=你的MySQL密码
```

> 密码通过 `.env` 文件配置，已加入 `.gitignore`，避免硬编码。其他数据库连接参数可在 `config.py` 中修改。

### 3.3 启动应用

```bash
..\.venv\Scripts\python.exe main.py
```

首次启动时，应用会自动创建 `smart_file_manager` 数据库及全部数据表，无需手动执行SQL。

## 4. 功能说明

应用包含7个主要标签页：**扫描管理**、**分类管理**、**文件搜索**、**操作历史**、**回收区**、**标签管理**、**系统设置**。

### 4.1 扫描管理

**用途**：扫描指定目录，将文件信息录入数据库。

**操作步骤**：
1. 点击 **浏览...** 按钮选择要扫描的目录
2. 根据需要勾选选项：
   - **递归扫描子目录**：扫描所有子文件夹中的文件
   - **计算文件哈希**：计算SHA256哈希值，用于后续去重（大文件较耗时）
   - **扫描后自动分类**：扫描完成后自动对文件进行智能分类
   - **提取文件元数据**：提取图片EXIF信息、PDF属性等
3. 点击 **开始扫描**，等待进度条完成
4. 扫描完成后，底部表格会显示所有已配置的扫描目录

**增量扫描**：对已扫描过的目录再次扫描时，只处理新增和修改过的文件，跳过未变化的文件。

### 4.2 分类管理

**用途**：查看文件分类结果，执行批量操作。

**界面布局**：
- **左侧**：分类导航树，包含三个维度：
  - 按类型（图片/文档/视频/音频/压缩包）
  - 按日期（如 2024年03月）
  - 按关键词（工作/学习/生活/项目等）
- **右侧**：点击左侧分类后，显示对应的文件列表

**批量操作**：
1. 在文件列表中多选文件（按住 Ctrl 点击）
2. 点击 **批量重命名**：按模板格式统一命名（默认：日期_类型_原名）
3. 点击 **批量移动**：选择目标目录，将文件移动到指定位置

**重新分类**：点击顶部 **重新分类所有文件** 可清除旧分类并重新执行全量分类。

### 4.3 文件搜索

**用途**：按多种条件检索数据库中的文件。

**搜索条件**：
| 条件 | 说明 |
|------|------|
| 文件名 | 关键词模糊匹配，支持中英文 |
| 类型 | 下拉选择：图片/文档/视频/音频/压缩包 |
| 最小大小 | 单位 KB |
| 最大大小 | 单位 MB，0 表示不限 |
| 重复文件 | 筛选仅重复/非重复文件 |
| 修改时间 | 勾选"启用日期筛选"后指定时间范围 |

输入条件后点击 **搜索** 或按回车键执行。搜索结果显示文件名、路径、类型、大小、修改时间、哈希值（前16位）和是否重复。

### 4.4 操作历史

**用途**：查看所有文件操作记录，执行撤销还原。

**筛选条件**：
- 操作类型：扫描/重命名/移动/删除/分类/去重/还原
- 时间范围：起止日期

**撤销操作**：
- **单条撤销**：点击每行末尾的 **撤销** 按钮
- **撤销选中操作**：多选后点击底部 **撤销选中操作** 按钮
- **撤销选中批次**：选中同一批次的记录，点击 **撤销选中批次** 一键还原整个批量操作

撤销后文件会恢复到操作前的状态（路径/文件名），操作状态标记为"已撤销"。

> 注意：永久删除的文件无法撤销。如果原路径已被其他文件占用，撤销会失败。

### 4.5 回收区

**用途**：管理已删除的文件，提供恢复或永久清除功能。

**工作原理**：删除文件时，文件会被移动到应用本地的 `.trash/` 目录（而非系统回收站），确保应用内可恢复。

**界面说明**：
- 表格显示已删除文件的文件名、原路径、大小、删除时间
- 支持分页浏览，每页 100 条
- 底部显示文件总数和本页占用空间

**操作说明**：
| 操作 | 说明 |
|------|------|
| **恢复** | 将文件从回收区移回原始路径，状态恢复为 active |
| **永久删除** | 删除回收区副本，状态标记为 purged，不可恢复 |
| **清空回收区** | 批量永久删除所有已删除文件的回收区副本 |
| **恢复/删除选中** | 多选后批量操作 |
| **刷新** | 重新加载回收区数据 |

> 清空回收区后，数据库记录仍保留（状态为 purged），但磁盘上的回收区文件已被清除。

### 4.6 标签管理

**用途**：独立的标签系统，为文件添加自定义标签，便于筛选和管理。

**界面布局**：
- **左侧**：标签云，标签颜色和字体大小根据关联文件数量自动变化
- **右侧**：选中标签后显示关联文件列表

**操作说明**：
| 操作 | 说明 |
|------|------|
| **新建标签** | 创建独立标签，不需要选中文件 |
| 标签云 → 点击 | 筛选该标签关联的文件 |
| **打标签** | 选中文件后输入标签名添加 |
| **移除标签** | 从选中文件中移除指定标签 |
| **删除标签** | 删除标签及其所有关联 |

### 4.7 系统设置

包含5个配置页面：

#### 通用设置
- 日志级别：DEBUG / INFO / WARNING / ERROR

#### 扫描设置
- 默认递归扫描子目录
- 是否包含隐藏文件
- 哈希算法：SHA256 / MD5
- 计算哈希的最大文件大小（超过此值跳过哈希计算）

#### 分类规则
管理智能分类使用的关键词规则：
- 点击 **添加规则** 输入规则名称、匹配模式（关键词用 `|` 分隔）、目标分类
- 选中规则后点击 **删除选中** 移除规则

默认内置4条规则：

| 规则名 | 匹配关键词 | 目标分类 |
|--------|-----------|--------|
| 工作文件 | 报告\|会议\|方案\|合同\|发票\|report\|meeting\|invoice | 工作 |
| 学习资料 | 笔记\|课件\|作业\|论文\|note\|homework\|thesis | 学习 |
| 生活照片 | 照片\|旅游\|美食\|photo\|travel\|food | 生活 |
| 项目文件 | 代码\|设计\|需求\|测试\|code\|design\|test | 项目 |

#### 重命名模板
自定义文件重命名格式，可用变量：

| 变量 | 含义 | 示例 |
|------|------|------|
| `{date}` | 文件修改日期 | 20240315 |
| `{time}` | 文件修改时间 | 143025 |
| `{type}` | 文件类型中文名 | 图片 |
| `{original_name}` | 原始文件名（不含扩展名） | 风景照 |
| `{ext}` | 文件扩展名 | .jpg |

默认模板：`{date}_{type}_{original_name}` -> `20240315_图片_风景照.jpg`

#### 去重策略
- **保留最新文件**：保留修改时间最新的副本
- **保留最旧文件**：保留修改时间最早的副本
- **保留路径最短**：保留目录层级最浅的副本
- **手动选择**：每组重复文件手动确认

## 5. 支持的文件类型

| 分类 | 扩展名 |
|------|--------|
| 图片 | jpg, jpeg, png, gif, bmp, webp, svg, ico, tiff, tif |
| 文档 | pdf, doc, docx, txt, xls, xlsx, ppt, pptx, csv, rtf, odt, md |
| 视频 | mp4, avi, mkv, mov, wmv, flv, webm, m4v, rmvb, rm |
| 音频 | mp3, wav, flac, aac, ogg, wma, m4a |
| 压缩包 | zip, rar, 7z, tar, gz, bz2, xz |

不在上述列表中的文件归类为"其他"。

## 6. 数据库表结构

| 表名 | 用途 |
|------|------|
| files | 文件索引（路径、名称、类型、大小、哈希、时间戳） |
| file_metadata | 元数据（图片EXIF、PDF属性、视频信息） |
| file_classifications | 分类记录（类型/日期/关键词维度） |
| operation_history | 操作历史（重命名/移动/删除/还原记录） |
| scan_directories | 扫描目录配置 |
| classification_rules | 分类规则配置 |
| system_settings | 系统设置键值对 |
| tags | 标签定义 |
| file_tags | 文件与标签的关联关系 |

## 7. 日志

日志文件位于 `smart-file-manager/logs/app.log`，单文件最大10MB，保留5个备份。

## 8. 常见问题

**Q: 启动时提示数据库连接失败？**
A: 确认MySQL服务已启动，检查 `config.py` 中的主机、端口、用户名和密码是否正确。

**Q: 扫描大目录很慢？**
A: 取消勾选"计算文件哈希"可显著提速。哈希计算需要读取完整文件内容，大文件耗时较长。也可在设置中调低"最大哈希文件大小"。

**Q: 撤销操作失败？**
A: 可能原因：文件已被手动移动/删除、原路径被其他文件占用。检查操作历史中的错误信息。

**Q: 如何添加新的文件类型识别？**
A: 编辑 `config.py` 中的 `FILE_TYPES` 字典，在对应分类下添加扩展名即可。

**Q: 回收区清空后文件还在？**
A: 清空回收区会永久删除磁盘上的回收区副本，并将状态标记为 purged。清空后刷新列表应显示“共 0 个文件”。如果仍有显示，请确认应用已更新到最新版本。
